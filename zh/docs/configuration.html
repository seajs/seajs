config

您可以使用 config 方法配置装载器的选项。 

seajs.config({
  base: 'http://path/to/libs/',
  alias: {
    'app': 'path/to/app/'
  },
  map: [
    ['http://example.com/', 'http://localhost/']
  ],
  charset: 'utf-8',
  timeout: 20000,
  debug: 0
});

可配置选项如下：

  • base 基本路径

    装载器在试图加载顶级模块标识时，将使用基本路径。详情请参阅 顶级路径 章节。

  • alias 别名

    当模块标识很长的时候，您可以使用 alias 以简化代码。 

    seajs.config({
  alias: {
    'app': 'path/to/app',
    'jquery': 'jquery/1.6.2/jquery'
  }
});

    a.js:

    define(function(require, exports, module) {
    var $ = require('jquery');
    // parsed to: require('jquery/1.6.2/jquery');

    var biz = require('app/biz');
    // parsed to: require('path/to/app/biz');
});
  • map 映射

    此配置项可以用来将某个文件迎映射到另一个。用来在线除虫十分方便。更多信息，请看 映射插件 章节。

  • charset 字符集

    获取模块文件时，<script> 标签的 charset 属性值。

  • timeout 超时

    装载器等候一个脚本的反馈的时间，单位毫秒，超时即认为不可用。默认是 20000 （20秒）。

  • debug

    如果 true，装载器会使用 console.log 输出所有的警告与错误。 默认是 false，装载器只会抛出异常。

    另外，您还可以将 debug 的值设为 2。如此， 每个脚本链接会加上唯一的时间戳。这在测试的时候很有用，可以防止浏览器使用缓存的版本。

noConflict

为了避免冲突，或者定制全局命名空间以符合自己的口味，您可以使用 noConflict 方法以获取 seajs 变量的初始状态值，并将其引用返回。 

var myLoader = seajs.noConflict();
myLoader.use('./main');

/* main.js */
define(function(require, exports, module) {
  // snip...
});

如果有必要，您还可以通过给该方法传递 true 作参数，以释放 define 方法。 很少会有这么做的必要。请三思而后行。 

var myLoader = seajs.noConflict(true);
myLoader.use('./main');

/* main.js */
myLoader.define(function(require, exports, module) {
  // snip...
});
