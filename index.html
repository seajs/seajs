A Module Loader for the Web

SeaJS 是一个适用于 Web 浏览器端的模块加载器。使用 SeaJS，可以更好地组织 JavaScript 代码。

也许你已注意到右边的美图：蓝天碧海之间，沙滩爱侣相伴。无论生活还是工作，我们都期望能享受如此良辰美景。SeaJS 在努力让我们的代码生涯变得更美好。

快速入门

在 SeaJS 的世界里，一个文件就是一个模块。所有模块都遵循 CMD 规范，我们可以像在 Node 环境中一样来书写模块代码： 

define(function(require, exports, module) {
  var $ = require('jquery')

  exports.sayHello = function() {
    $('#hello').toggle('slow')
  }
})

将上面的代码保存为 hello.js，然后就可以通过 SeaJS 来加载使用了： 

seajs.config({
  alias: {
    'jquery': 'http://modules.seajs.org/jquery/1.7.2/jquery.js'
  }
})

seajs.use(['./hello', 'jquery'], function(hello, $) {
  $('#beautiful-sea').click(hello.sayHello)
})

赶紧点击右上角的美图，与模块世界打个招呼。

核心特性

无论是中小型站点，还是大型复杂应用，使用 SeaJS，都可以让我们的工作变得更轻松愉悦。SeaJS 具有以下核心特性：

  • 代码格式简单一致：所有模块代码，都按照 CMD 规范来书写。
  • 自动管理依赖：无论是多层级依赖还是循环依赖，SeaJS 都能帮你处理好。
  • 异步并行加载：异步无阻与并行加载，可以让页面性能达到默认优化。
  • 拥有丰富的插件：SeaJS 的可扩展性非常好，目前已有近 10 个插件。
  • 对调试非常友好：通过 debug 插件，可以实现在线调试等功能。

SeaJS 带来的最大好处是：提升代码的可维护性。上面的每一项特性，在接下来的文档中都会有详细阐述。除了这些核心特性，SeaJS 还具有不少非常实用的功能，比如可用于 mock 或快速 fix bug 的 seajs.modify 等方法。

SeaJS 追求的是更简单、更自然的代码书写和组织方式。
